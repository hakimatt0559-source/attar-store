# عطارة مغنية الشطية — الموقع الجديد

موقع تجريبي (المرحلة الأولى) مبني بـ React + Vite + Tailwind CSS، بدون أي شعار أو قيود منصات خارجية.

## ✅ الحالة الحالية
- الصفحة الرئيسية (تصنيفات + عرض اليوم + الأكثر طلباً) ✅
- صفحة تفاصيل المنتج + خوارزمية "قد يعجبك أيضاً" ✅
- صفحات مبدئية (التصنيفات، الطلبات، حسابي، السلة) — قيد الإنشاء لاحقاً
- قاعدة بيانات حقيقية (Supabase) — لسه مش متصلة، الموقع شغال ببيانات وهمية مؤقتاً

## 📱 خطوات الرفع على GitHub من الهاتف

1. افتح تطبيق أو موقع GitHub من الهاتف.
2. اذهب لحساب الـ repo الفارغ الجديد اللي عملته.
3. استخدم خيار "Upload files" (رفع ملفات) وارفع كل الملفات والمجلدات بنفس الهيكل الموجود هنا (حافظ على أسماء المجلدات src, public بالظبط).
4. اكتب رسالة commit زي: "أول نسخة من الموقع الجديد" واضغط Commit changes.

## 🚀 خطوات النشر (Deploy) مجاناً

**الطريقة الموصى بها: Cloudflare Pages**

1. افتح [pages.cloudflare.com](https://pages.cloudflare.com) وسجل دخول (أو أنشئ حساب مجاني).
2. اختر "Create a project" → "Connect to Git".
3. اختر الـ repo اللي رفعته على GitHub.
4. في إعدادات البناء (Build settings):
   - Build command: `npm run build`
   - Build output directory: `dist`
5. اضغط Save and Deploy.
6. بعد دقيقة أو اتنين، هيديك رابط مجاني زي: `attar-store.pages.dev` — بدون أي شعار.

**ملاحظة:** كل مرة تعدل الكود وترفعه على GitHub (commit جديد)، Cloudflare هينشر النسخة الجديدة تلقائياً.

## 🗄️ ربط قاعدة البيانات (Supabase) — خطوة قادمة

1. اعمل حساب على [supabase.com](https://supabase.com) (مجاني).
2. أنشئ مشروع جديد.
3. من إعدادات المشروع (Settings → API)، هتلاقي:
   - `Project URL`
   - `anon public key`
4. في Cloudflare Pages، روح لإعدادات المشروع → Environment variables وضيف:
   - `VITE_SUPABASE_URL` = الرابط بتاعك
   - `VITE_SUPABASE_ANON_KEY` = المفتاح بتاعك
5. أعد النشر (Redeploy).

بمجرد ما تضيف هذه المتغيرات، الموقع هيبدأ يسحب البيانات الحقيقية من Supabase تلقائياً بدل البيانات الوهمية — الكود مجهز لده من الأول (شوف `src/lib/products.js`).
